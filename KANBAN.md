# План работ — Soul Knight 3D

## В разработке

- [ ] Проверить матч на 2–4 игрока через Render: задержка, потеря связи, повторный вход.
- [ ] Перенести полоски здоровья врагов из HTML в Three.js, чтобы уменьшить нагрузку на браузер.

## Следующие задачи

- [ ] Отправлять игроку врагов и предметы только из текущей и соседних комнат.
- [ ] Отправлять содержимое инвентаря только владельцу инвентаря.
- [ ] Сохранять PvE-прогресс матча при кратком разрыве соединения.

## В очереди

- [ ] Настоящие GLB-модели игрока, врагов, оружия и окружения.
- [ ] Анимации: idle, run, shoot, reload, melee, hit, death.
- [ ] Настоящий магазин с валютой вместо комнаты снабжения.
- [ ] Редкости и модификаторы предметов.
- [ ] Задания и мета-прогрессия.
- [ ] Матчмейкинг и приватные лобби.
- [ ] Более строгая проверка попаданий на сервере.
- [ ] Лимиты частоты действий и журнал подозрительных действий.
- [ ] Публичная страница правил и политики хранения данных.

## Готово

- [x] HttpOnly-сессии и авторизация.
- [x] Лобби и вход по коду.
- [x] Базовые PvE и PvP режимы.
- [x] PostgreSQL для опубликованной версии и локальная база для разработки.
- [x] Кэш 3D-объектов по id.
- [x] Переход в PvE-комнату после подтверждения всех живых игроков у двери.
- [x] Передача полной картины мира только при входе и отправка только изменений во время матча.
- [x] Защита от одновременного запуска нескольких серверных циклов.
